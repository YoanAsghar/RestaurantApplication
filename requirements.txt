# Restaurant Application Backend Roadmap (Minimal API)

## Phase 1: API Foundation and Database Models

- [X] **Configure Project for Minimal API:**
  - Ensure `Program.cs` is set up for Minimal APIs. (No `AddControllers` or `MapControllers`).
  - Configure a CORS policy to allow requests from the frontend development server.

- [X] **Setup Database Models:**
  - Create a `Product` model class (e.g., in a `Models` folder) with properties like `Id`, `Name`, `Description`, `Price`, `ImageUrl`, `Category`.
  - Create a `Category` model class (`Id`, `Name`).
  - Create `Order`, `OrderDetail`, and `ShoppingCartItem` models.
  - Update `ApplicationDbContext` to include `DbSet` for all new models.
  - Add a new database migration (`dotnet ef migrations add AddAppModels`) and update the database (`dotnet ef database update`).

## Phase 2: API - User Authentication and Roles

- [ ] **Configure Identity for API:**
  - Set up ASP.NET Core Identity to use token-based authentication (e.g., JWT) instead of cookies.
  - Add and configure JWT bearer authentication services.

- [ ] **Create Authentication Endpoints:**
  - In `Program.cs` or a dedicated routes file, map the authentication endpoints.
  - Map a `POST /api/register` endpoint to create a new user.
  - Map a `POST /api/login` endpoint to authenticate a user and return a JWT.

- [ ] **Seed Admin User and Roles:**
  - Create an "Admin" and "Customer" role programmatically.
  - Write a database seeding utility to create an initial admin user and assign the "Admin" role.

## Phase 3: API - Admin Product Management

- [X] **Create Admin Product Endpoints:**
  - Map admin-only endpoints, securing them with `.RequireAuthorization("Admin")`.
  - `POST /api/products` endpoint for creating a new product.
  - `PUT /api/products/{id}` endpoint for updating an existing product.
  - `DELETE /api/products/{id}` endpoint for deleting a product.

## Phase 4: API - Public Storefront

- [X] **Add Public Product Endpoints:**
  - Map public endpoints that do not require authorization.
  - `GET /api/products` endpoint to fetch a list of all products.
  - `GET /api/products/{id}` endpoint to fetch a single product by its ID.

## Phase 5: API - Shopping Cart and First-Purchase Discount

- [ ] **Create Cart Endpoints:**
  - Map cart endpoints, securing them for authorized customers.
  - `GET /api/cart` endpoint to get the current user's cart.
  - `POST /api/cart/items` endpoint to add an item to the cart.
  - `DELETE /api/cart/items/{itemId}` to remove an item from the cart.

- [ ] **Implement Discount Logic:**
  - When a user registers, set a flag on their user profile (e.g., `IsFirstPurchase = true`).
  - The `GET /api/cart` endpoint logic should check this flag and apply a discount to the total if applicable.
  - The checkout process will be responsible for flipping this flag to `false`.

## Phase 6: API - Newsletter and Emailing

- [ ] **Create Newsletter Endpoints:**
  - Map a public `POST /api/newsletter/subscribe` endpoint that accepts an email address.
  - The endpoint should save the email to a `NewsletterSubscriptions` table.

- [ ] **Implement Email Service:**
  - Configure an email sending service (e.g., SendGrid, MailKit) and an `IEmailSender` implementation.
  - Use the service to send a "Thank you for subscribing" email from the `/subscribe` endpoint.

## Phase 7: API - Order and Sales Management

- [ ] **Create Order Endpoints:**
  - Map an `POST /api/orders/checkout` endpoint (authorized) that creates an `Order` and `OrderDetail` record from the user's cart and then clears the cart. This is where the `IsFirstPurchase` flag gets updated.
  - Map a `GET /api/orders` endpoint (authorized for Admins) to list all historical orders for the sales management dashboard.
